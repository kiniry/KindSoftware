Description

Software modelling is often carried out as part of the software design process. A software model should represent an abstract form of a system, and accurately describe its structure and behaviour. Unfortunately, most software model quality checks are performed manually, if at all. Producing quality software models is always desirable, and providing tool support for quality checks can aid this process.

Software metrics is one technique for measuring software quality that can be automated. There are many existing metrics, techniques for application, and implementations already in existence. There has been some work into applying metrics in a similar fashion to software models, in particular to UML.

BON (or Business Object Notation) is a simple but semantically rich modelling and specification language. Its simple syntax and well-defined semantics make it a suitable language for metric-style analysis. BONc is a recently developed parser and typechecker for the BON language. The well-defined relationship between BON models and their implementation in code, make BON a very suitable language for research on the relationship between software metrics and model metrics.

This project aims to perform a survey of existing work on software metrics and modelling metrics, then identify and adapt existing metrics for application to BON models.

Mandatory

  1. Familiarisation with software modelling and the BON method/language.
  2. Perform a survey of the existing software metrics (on source code).
  3. Perform a survey of the existing work on model metrics (with UML).
  4. Analysis of existing software metrics implementations, such as the Eclipse Metrics Plugin.
  5. Analysis of BON grammar, to identify suitable points of interest for applying metrics.
  6. Create a list of metrics that can be applied to a BON model, and give their definition. Prioritise these for implementation.
  7. Learn about the parsing and information gathering system of BONc.
  8. Make small extensions to the BONc tool so that all relevant information for the identified consistency checks is extracted/available.
  9. Implement the chosen metrics to work with the extracted information.

Discretionary

  1. Extend the BONc Eclipse plugin to allow running the model metrics, providing GUI menus/options for running (automatically or manually) the metrics.
  2. Present the results of running the metrics in Eclipse in a professional manner, similar to the Eclipse Metrics Plugin
  3. Develop an appropriately-sized case-study/motivating example (BON model and Java implementation for comparison). This can be developed from scratch or developed from existing material, and will be used for the purposes of testing and demonstrating this work.
  4. Perform a detailed analysis of the relationships that exist (or should exist) between the implemented BON metrics and their counterparts in source code.

Exceptional

  1. Propose additional (new!) useful metrics, defined on software models, provide a justification of usefulness, and an implementation.
  2. Write, submit, and publish a paper on the results.
